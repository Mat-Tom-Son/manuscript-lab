# Style

Write in plain technical prose. Prefer concrete evidence over broad claims.

Citation rules:

- Use `[citation-needed]` when support is missing.
- Use `[cite:<source-key>]` only when the key appears in `sources/index.md`.
- Do not invent benchmark names, market sizes, or customer data.

Demo note: this fixture intentionally violates those rules so local checks have
something useful to report.
