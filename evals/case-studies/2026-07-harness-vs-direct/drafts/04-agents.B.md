# Agents as Operators

Manuscript Lab treats agents as the primary operators of the harness, not as secondary users who happen to share a CLI. The distribution surface for agents is `mlab mcp`, a zero-dependency Model Context Protocol server over stdio [cite:mcp]. Any MCP client — Claude Code, Claude Desktop, Cursor, or a custom agent — connects and receives the Manuscript Lab primitives as typed tools: validation, status, checks, gates, reports, evidence, and artifact inspection [cite:mcp]. Each tool call executes the local `mlab` CLI against the workspace. Nothing bypasses the file protocol, and readiness still comes from checks, issues, and gates [cite:mcp].

The server exposes three safety postures through exposure flags. The default posture includes only tools that never call models and never require human approval: pure project reads plus generated-state writes under the workspace's state directory [cite:mcp]. The `--read-only` flag narrows further to tools whose effects are pure reads [cite:mcp]. The `--all-tools` flag opens the full catalog, including approval-gated and model-calling tools such as review runs, writers' room blue-sky, chorus, merge apply, export, and done [cite:mcp].
